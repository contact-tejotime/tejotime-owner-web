1. Executive Summary

TejoTime is a multi-tenant queue + booking + CRM platform for small service businesses (salons, barbershops, and similar). Each store = one business tenant with a public microsite, owner mobile app, and operational data stored in a shared backend.

Today: The admin panel handles store provisioning (create/edit microsites) and a basic All Stores list. It has no analytics.
Goal: Transform the admin panel into a platform operations center with platform-wide dashboard, enhanced store directory with live metrics, and per-store deep dive into customers, visits, bookings, and revenue trends.

All required data already exists in the database (customer, visit, appointment, queue_entry). New admin-scoped API endpoints are needed to expose it cross-tenant.

7. Navigation & UX Principles

  1. 3-second rule — Every page opens with 4–6 headline KPIs above the fold.
  2. Today → Trend → Detail — Overview first; tables for drill-down.
  3. Same vocabulary as owner app — Visits, Revenue, Appointments, Customers.
  4. Money in ₹ — Format paise consistently (₹1,840 or ₹18.4k).
  5. Status colors — Green = active/completed, amber = waiting, red = no-show/cancelled.
  6. Period comparison — Show "+12% vs last week" on KPI cards when deltas are available.
  7. Empty states — Contextual messages when a store has no data yet.

9. Implementation Phases

Phase 1 — 1–2 weeks

Quick Wins

  • Enhance /stores table with metrics columns and clickable rows
  • Store overview tab with today + all-time KPIs
  • Backend: GET /admin/businesses/:id/analytics (basic aggregates)
Phase 2 — 2–3 weeks

Deep Analytics

  • Platform /dashboard with cross-store KPIs
  • Revenue + visit trend charts per store
  • Customer list page with search and filters
  • Visit ledger page
Phase 3 — Later

Advanced

  • Booking funnel, peak hours heatmap, customer segmentation
  • CSV export for visits and customers
  • Period comparison deltas ("+12% vs last week")
  • daily_metric rollup table for performance at scale
