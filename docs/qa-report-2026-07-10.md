## Bugs found & fixed

1. **Zombie queue entries inflated "Active in queue"** — two `in_service` entries from 4 Jul (`Darshil`, `hjb`, both `staff_id=null`) were invisible in the owner queue UI but counted in owner `activeCount` and admin `today.activeQueue` (showed 3 when the real queue had 1). Root cause: the stale-cleanup cron only cancelled `waiting` entries — abandoned `in_service` entries lived forever. **Fixed:** `backend/src/jobs/scheduler.ts` now cancels both `waiting` and `in_service` entries older than `TICKET_ABANDON_HOURS` (4h). The two zombie rows were repaired (marked `cancelled`).
2. **Billing page showed fake data as real** — every premium store showed "Next invoice: 1 Aug 2026 · Status: Paid" from `STATIC_BILLING`. **Fixed:** backend `GET /admin/businesses?withMetrics=1` now returns the real `subscription.status` per store; the billing page renders it as a badge (Active/Trialing/Past due/Canceled) for all stores; the "Next invoice" column was removed (no payments backend exists — `PAYMENTS_ENABLED=false`); `STATIC_BILLING` deleted.
3. **Dead code removed** — `STATIC_ATTENTION_ITEMS` (always empty) deleted from `static-data.ts` and `dashboard/page.tsx`; the real quiet-stores logic remains.
